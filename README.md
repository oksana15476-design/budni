# Будни — лендинг кадрового агентства

Direct-response лендинг кадрового агентства «Будни»: массовый и точечный подбор персонала, executive search. Дизайн — финальная итерация хэндоффа Claude Design (структура v4 «HR-OS» + розово-голубая палитра v2).

Статика без сборки: семантический HTML, CSS, ванильный JavaScript. Self-hosted шрифт Golos Text, JSON-LD (EmploymentAgency + FAQPage), Open Graph + og-image, sitemap/robots, honeypot-защита форм.

Сайт: **https://oksana15476-design.github.io/budni/** (GitHub Pages, ветка `main`).

## Структура

```
index.html       — страница целиком (все секции, мета, JSON-LD, SVG-спрайт)
privacy.html     — политика обработки персональных данных (152-ФЗ)
css/styles.css   — дизайн-система и стили
js/main.js       — интерактив + CONFIG (приём заявок, Метрика)
fonts/           — Golos Text (woff2, self-hosted)
assets/og-image.png — превью для соцсетей/мессенджеров (1200×630)
robots.txt, sitemap.xml
```

## ⚙️ Что нужно заполнить, чтобы сайт «включился»

Все настройки — в блоке `CONFIG` в начале `js/main.js`:

**1. Приём заявок** (сейчас заявки показывают «отправлено», но никуда не уходят) — один из вариантов:
- `LEAD_ENDPOINT` — URL, принимающий POST с JSON. Самый простой путь: [Formspree](https://formspree.io) — регистрируетесь, создаёте форму, вставляете URL вида `https://formspree.io/f/XXXXXX`.
- `TELEGRAM: { botToken, chatId }` — заявки приходят сообщением в Telegram. Создайте бота через @BotFather, узнайте chat_id через @userinfobot. **Важно:** токен в статике виден всем посетителям — заведите отдельного бота только под заявки (максимальный риск — спам в этот чат).

**2. Яндекс.Метрика** — `METRIKA_ID: 12345678` (номер счётчика с metrika.yandex.ru). Цели уже прошиты: `lead_quiz`, `lead_calculator`, `lead_contact` — создайте их в Метрике как JavaScript-события.

**3. Контакты** — когда будете готовы публиковать телефон/Telegram: скажите Claude или добавьте в шапку (`.header-link`), футер (колонка «Контакты») и sticky-бар (кнопка «Написать» с `https://t.me/...`).

**4. Реквизиты** — в `privacy.html`, блок «Оператор персональных данных»: ФИО ИП, ИНН, ОГРНИП. Их же стоит добавить в футер главной — это доверие для B2B.

**5. Свой домен** — купите домен, в настройках репозитория Settings → Pages → Custom domain. После переезда замените `oksana15476-design.github.io/budni` на домен в: `index.html` (canonical, og:url, og:image, twitter:image), `robots.txt`, `sitemap.xml`.

## Блок «Кто ведёт проект» (рекомендуется)

Лучшая правка для доверия — реальное лицо. Когда будут фото и факты, попросите Claude добавить секцию основателя: имя, фото, опыт (лет в подборе, сколько вакансий закрыто), 2–3 принципа работы. Шаблон секции согласован с дизайн-системой — добавляется за один заход.

## Запуск локально

